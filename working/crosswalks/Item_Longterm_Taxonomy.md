# Item Longterm Taxonomy

## Principle

지금은 흩어진 아이템 후보를 모으는 단계이고, 정식 아이템 백과의 대분류는 나중에 확장한다.

핵심은 `지금 당장 완성`이 아니라 `나중에 크게 늘어나도 안 꼬이는 구조`를 먼저 고정하는 것이다.

## Planned Top Layers

1. 일반 장비
2. 성장형 아이템
3. 에고형 아이템
4. 설정집 산재 아이템
5. 계보형 전설 아이템군
6. 세트형 장비군
7. 컬렉션형 보물군

## 1. 일반 장비

- 무기
- 방어구
- 악세서리
- 유물
- 소모성 특수 장비

이 층은 현재 `Item_Candidate_Register.md`에서 가장 먼저 모이는 기본 축이다.

## 2. 성장형 아이템

- 사용자와 함께 강해지는 장비
- 단계별 각성, 진화, 해방, 변이 기록이 필요한 장비
- 특정 액트 또는 관계 변화와 함께 형태가 변하는 장비

필수 필드:

- 성장 단계
- 성장 조건
- 각성 계기
- 사용자와의 관계
- 최종 형태 존재 여부

## 3. 에고형 아이템

- 자의식, 인격, 의지, 기억, 목소리를 가진 아이템
- 사용자와의 관계가 계약, 갈등, 공생, 잠식으로 전개될 수 있는 아이템

필수 필드:

- 에고 유무
- 에고 성향
- 의사소통 방식
- 사용자와의 관계 상태
- 독자 행위 가능 여부

## 4. 설정집 산재 아이템

- 인물 문서, 세력 문서, 지역 문서, 사건 문서에 흩어진 상태로 먼저 발견되는 항목
- 아직 정식 백과 문서가 없는 항목
- 당분간은 이 층에 보류한다

이 층은 임시 흡수층이다.

## 5. 계보형 전설 아이템군

- 숫자나 명칭으로 묶이는 전설 무기군
- 단일 아이템보다 `같은 계보에 속한 묶음`이 먼저 중요한 경우
- 소유자 계승, 분실, 파손, 위조, 결번 여부를 따로 관리한다

### 7대 보검

- 세계관 내 상위 상징 무기군
- 각 보검의 명칭, 계보, 현재 소유자, 분실 여부, 파손 여부를 따로 기록

### 11대창

- 상동
- 보검과 같은 규격으로 관리하되, 같은 층에 섞지 않고 독립 컬렉션으로 둔다

이 층은 예시가 더 늘어나도 같은 방식으로 수용한다.

- `N대 보검`
- `N대창`
- `N대 활`
- `왕가 계승 무구군`
- `봉인 무장군`

## 6. 세트형 장비군

- 전신 세트, 의식 세트, 기사단 제식 세트처럼 여러 부위가 묶여 의미를 갖는 장비군
- 개별 장비보다 세트 완성 효과, 착용 조건, 누락 파츠 여부가 중요하다

예시 수용 범위:

- 전신 세트
- 성직 세트
- 기사단 제식 세트
- 마도 공학 장비 세트
- 보스 드롭 세트

필수 필드:

- 세트명
- 파츠 목록
- 완성 조건
- 세트 효과
- 대표 사용자 또는 세력

## 7. 컬렉션형 보물군

### 보물 컬렉션 235

- 수집, 탐색, 분실, 전승, 소문, 위조품 여부가 중요한 축
- 한 번에 다 정리하지 않고 `번호 / 명칭 / 상태 / 위치 / 관련 인물` 기준으로 운영한다

### 전집 보물 무더기

- 아직 이름과 범위가 불안정한 묶음형 보물군
- 지금은 개별 아이템보다 `컬렉션 단위`로 먼저 관리한다

## Recommended Order Later

1. 일반 장비 안정화
2. 성장형 / 에고형 분리
3. 설정집 산재 아이템 흡수
4. 계보형 전설 아이템군 정리
5. 세트형 장비군 정리
6. 컬렉션형 보물군 정리

## Do Not Do

- 성장형과 에고형을 같은 표로 뭉개지 않는다
- 계보형 전설군, 세트형 장비군, 컬렉션형 보물군을 일반 장비와 같은 우선순위로 섞지 않는다
- 현재 산재 아이템을 억지로 전설급 아이템 체계에 끼워 맞추지 않는다
