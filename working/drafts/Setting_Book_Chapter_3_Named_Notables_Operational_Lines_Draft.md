# Setting Book Chapter 3 - People Worth Seeking Draft

이 장은 `15번` 축을 설정집 본문에서 어떻게 읽을지 정리하는 1차 초안이다.

핵심은 `14번` 영웅 백과와
`찾아가야 할 이름들` 본문 축을 맞추고,
`15번` operational lines handoff를 분리해서 읽는 것이다.

`15번`은 약한 영웅 창고가 아니다.
세계 안에서 찾아갈 이유가 있는 이름 있는 명사형 인물과,
세력 작동을 보여주는 실무층을 회수하는 축이다.

## 3.0 Reading Rule

`15번`은 두 레이어로 운영한다.

| Layer | Use |
|---|---|
| `People Worth Seeking` | 현자, 장인, 기록자, 감정사, 연금술사, 공방주처럼 이름이 알려진 명사형 인물 |
| `Operational Lines` | 항만, 경매, 금융, 첩보, 위조, 운반, 용병 운영처럼 조직 작동을 보여주는 실무층 |

아래 신호가 강하면 바로 `15번`으로 내리지 않는다.

- `SS / S / A급`
- Act 중심성
- 전설 영웅록
- 독립 14 파일
- 조직 최고위가 곧 중심 영웅처럼 작동하는 경우

이 경우 먼저 `verify_before_15` 또는 `source_check_hold`로 둔다.

## 3.1 Current Mainline State

현재 본선은 신규 후보 확장이 아니라
`mainline reference carryover` 유지다.

설정집 본문에서 현재 상태는 아래처럼 읽는다.

| State | Meaning | Setting Book Use |
|---|---|---|
| `stable_15_workset` | 현재 닫힌 stable reference set | 확정 본문이 아니라 reference-only subset으로만 사용 |
| `source_check_hold` | 근거는 있으나 14/15 경계나 충돌 확인 전 | 본문 확정 금지, hold reference split 안에서만 유지 |
| `deferred_expansion_hold / hold reference split` | 후기 확장축 / 범대륙 등 본선 밖 보류 | mainline 밖 appendix 또는 hold note로 유지 |
| `hold reference split` | stable set과 분리된 보류 reference | stable 후보와 섞지 않음 |

## 3.2 stable_15_workset

현재 닫힌 stable reference subset은 아래 세 명이다.

1. `울프가르 드래곤포지`
2. `에리온 드라코비스`
3. `오그마`

설정집 safe read:

| Name | Section 8 Anchor | Safe Use |
|---|---|---|
| `울프가르 드래곤포지` | `크림슨 / 용의 후예 / 드래곤포지 공방` | 장인 / 대장장이 / 공방주 reference |
| `에리온 드라코비스` | `크림슨 / 엘드라칸 / 학술-전승층` | 현자 / 고대어 해석가 / 관리자 reference |
| `오그마` | `크림슨 / 엘드라칸 / 전승 보관층` | 전승 보관자 / 살아있는 도서관 reference |

주의:

- 이 셋은 Hard Canon 확정 인물로 승격하지 않는다.
- `14` 독립 파일 확인 전까지 grade / act caution을 유지한다.
- 상위 route anchor와 place lock을 섞지 않는다.
- `stable_15_workset`을 다시 열어 신규 인물 평가 흐름으로 만들지 않는다.

## 3.3 Hold Reference Split

현재 stable set과 분리해 읽는 주요 hold 후보는 아래다.

| Name | State | Safe Use |
|---|---|---|
| `엘다라` | `source_check_hold / hold reference split` | 에테르 정령연합 내부 전승층 얼굴. 정령연합 전체 14 확인 전 Hard Canon 금지 |
| `실비아` | `deferred_expansion_hold / hold reference split / name_collision_watch` | 키르케 계열 기록자 / 약제사 축. 범대륙 후기 확장이라 mainline 밖에서 유지 |
| `멜리산드르` | `source_check_hold` | 키르케 영약회 최고위. 14 crossover risk 때문에 dual review 필요 |

설정집 safe read:

- hold 후보는 본문에서 확정 인물처럼 쓰지 않는다.
- `엘다라`는 정령연합 내부 예외축으로만 둔다.
- `실비아`는 5대륙 본선 증거가 아니라 후기 확장축으로 둔다.
- hold 후보는 `stable_15_workset`과 같은 표에서 병합하지 않는다.

## 3.4 Continental Coverage

대륙별 coverage는 아래처럼 읽는다.

| Area | Current Read | Main Risk |
|---|---|---|
| `크림슨` | stable workset이 가장 잘 보임 | SS/S/A급, 영웅록, 공병대장/예언자 신호 |
| `에테르` | place-institution slot과 source_check_hold 후보가 많음 | 탑주, A/S급, Act 신호, 이름 드리프트 |
| `프로스트` | named holder보다 장소/기관 slot이 강함 | 전설/핵심 영웅 신호 |
| `해양` | 항해/연금/항구/암시장 명사 후보가 많음 | A급, 히어로급, 제독/단장, 이름 충돌 |
| `오벨리스크` | 기록/봉인/영혼/기억 명사 후보가 많음 | 전설 영웅록, 아이템/name collision, 조직 핵심 신호 |
| `범대륙 / 후기 확장` | hold reference split | 후기 증설 구역 과확정 위험 |

커버리지가 얇은 대륙에 후보를 억지로 만들지 않는다.

이름 있는 후보가 없으면
`Operational Lines`, `Institution Memory`, `Place Function` 쪽 reference backlog로 남긴다.

## 3.5 People Worth Seeking Guard

People Worth Seeking은 아래 기준을 통과해야 한다.

1. 세계 안에서 이름이 알려져 있다.
2. 중심 영웅은 아니지만 찾아갈 이유가 있다.
3. 세력 / 도시 / 조직 anchor가 명확하다.
4. 기능이 `현자 / 장인 / 기록자 / 감정사 / 연금술사 / 공방주`처럼 명사형이다.
5. `14번` 중심 영웅 신호가 강하면 먼저 hold한다.

금지:

- 14 중심 영웅을 15로 낮춰 넣지 않는다.
- A급/전설/Act 중심 인물을 단순히 명사형이라는 이유로 15에 고정하지 않는다.
- 이름 충돌 후보를 하나로 병합하지 않는다.
- 실무층 operational profile을 `People Worth Seeking` 승인 근거로 역수입하지 않는다.

## 3.6 Operational Lines Guard

Operational Lines는 유명 인물보다
조직이 실제로 어떻게 움직이는지를 보여주는 실무층이다.

예:

- 항만 운영.
- 경매.
- 금융.
- 첩보.
- 위조.
- 장물 처리.
- 운반.
- 용병 운영.
- 기록 보관.
- 보급 / 봉인 / 감시.

Operational profile card는 `3-1. Policy Guard`를 하위 권한선으로 유지한다.

설정집 본문에서는 operational profile 문장을 요약할 수는 있지만,
exact wording authority를 가져오지 않는다.

즉:

- `People Worth Seeking` card와 operational profile card를 같은 증거층으로 합치지 않는다.
- 상위 summary가 하위 `3-1. Policy Guard`를 재정의하지 않는다.
- `Port Authority / Black Auction / Gravewell / Counterfeit Workshop` 같은 대표 pair는 닫힌 subline 교차감사 샘플로만 읽는다.

## 3.7 Section 8 Anchor Rule

현재 stable / hold 후보는 모두 `Section 8` anchor를 가진다.

설정집에서 15 후보를 쓸 때는 아래 순서를 따른다.

1. `Section 8` 대륙 / 세력 / 도시 / 조직 anchor 확인.
2. structure label과 place pressure 분리.
3. route anchor와 place lock 분리.
4. 14/15 boundary 확인.
5. state label 확인.
6. lower-card authority 확인.

특히 `엘다라`는
`에테르 / 정령연합 / 루미라` route 후보로만 보존한다.
정령연합 전체 14 확인 전에는 `source_check_hold / hold reference split` 상태를 넘지 않는다.

## 3.8 Chapter Lock

이 장에서 고정하는 기준은 아래다.

- `15번`은 약한 영웅 창고가 아니다.
- `stable_15_workset`은 닫힌 reference set이다.
- `source_check_hold` 후보는 확정 본문으로 올리지 않는다.
- `deferred_expansion_hold / hold reference split`는 mainline 밖에서 유지한다.
- coverage가 얇은 대륙에 새 후보를 억지로 만들지 않는다.
- operational profile의 lower-card authority를 `People Worth Seeking` 승인 논리로 역수입하지 않는다.
- 다음 장인 `4. 이름 / 명칭 정규화`는 이 14/15 분리 기준을 유지한 채 이름 충돌과 표면명만 다룬다.
