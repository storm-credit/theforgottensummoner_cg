# Setting Book Hub Completion Checklist

## 왜 이 문서가 필요한가

허브 작업은
"허브 파일이 생겼다"와
"옵시디언에서 실제로 덜 헤맨다"가 다르다.

그래서 진행률을 감으로 올리지 않고,
어디까지 끝났고 무엇이 남았는지 체크리스트로 고정한다.

## 현재 판단 기준

앞으로 허브 진행률은 아래 다섯 층을 나눠서 본다.

1. 허브 파일 존재
2. 허브끼리 왕복 링크 존재
3. 각 허브 안에 대표 진입 문서가 명확함
4. 옵시디언에서 클릭 동선이 자연스러움
5. 허브 설명 문체와 기준이 통일됨

## 진행률 보는 법

이제부터는 두 가지로 나눠서 본다.

- 허브 골격 완성도
  - 허브 파일이 있느냐, 왕복 링크가 있느냐
- 허브 사용성 완성도
  - 옵시디언에서 실제로 덜 헤매느냐

현재 오케스트라 판단:

- 허브 골격 완성도: 높음
- 허브 사용성 완성도: 전보다 안정됐지만 아직 마감 전

즉, "허브가 있다"는 이유만으로 90% 가까이 올리면 안 되고,
"허브를 실제로 타고 다니기 편하다"까지 가야 높은 진행률로 본다.

## 지금 남은 허브 마감 일

1. 허브끼리 중복 없이 가장 짧은 경로로 이동되는지 재검토
2. 세력 허브 안에서 실제 세력별 세부 노트 자리를 더 또렷하게 만들기
3. 메인 허브의 첫 진입 동선을 실제 사용자 시점으로 한 번 더 재검토
