# Setting Book Chapter 5 Draft: Items, Relics, and Desire Structure

## Chapter Purpose

This chapter defines how the setting book should handle weapons, armor, accessories, relics, trade goods, growth items, ego items, legendary groups, and collectible treasures.

The goal is not to list every extracted item candidate immediately. The goal is to build an item framework that makes readers want to know, own, collect, fear, recover, or argue about the objects in the world.

An item entry should not begin as a performance table. It should begin as a story promise.

## Item Canon Layers

The current item material should be sorted into four layers before encyclopedia promotion.

| Layer | Meaning | Setting-Book Use |
| --- | --- | --- |
| Extracted candidate | A name or object-like phrase found in source material | Track only; do not treat as confirmed lore yet. |
| Confirmed object | A real item, artifact, trade good, or owned object with context | Eligible for summary or appendix mention. |
| Story-bearing item | An item with owner, use, origin, cost, conflict, or desire hook | Eligible for setting-book entry. |
| Signature relic | A story-bearing item with recurring symbolic or plot weight | Eligible for featured relic profile. |

Personal gear from character documents should not be automatically canonized as major relics. A sword in a hero sheet may be equipment, a growth item, a family heirloom, a symbolic burden, or extraction noise. Promotion depends on story function.

## Candidate Promotion Rules

An item can move toward a setting-book entry only when it has enough evidence.

Minimum evidence:

- It is confirmed to be an object rather than a technique, doctrine, monster, material word, location, or faction label.
- It has at least one owner, representative user, faction, source scene, or trade context.
- Variant names have been checked.
- It is not caught in a protected name collision without resolution.
- It has a reason to matter beyond raw power.

Required tracking fields:

- item name
- category: weapon, armor, accessory, relic, trade good, tool, vehicle, device, material, unknown
- owner or representative user
- faction or cultural context
- first source
- variant names
- status: candidate, duplicate candidate, needs merge, needs classification, ready for encyclopedia

## What Not to Promote Too Early

The setting book should avoid premature promotion in the following cases:

- Pure material terms such as gold, jewel, obsidian, or crystal unless they are named goods or relic substances.
- Mounts, bio-devices, or living constructs unless the setting decides they are treated as owned artifacts.
- Information devices, research tools, or magical apparatuses without story function.
- Magic names that sound like items.
- Family names, titles, or shield concepts using item-like terms.
- Repeated weapon names that may belong to different people.

The `Aegis` cluster is the clearest protected collision: it may refer to a person, family name, item, magic, shield doctrine, or symbol. Item use must not absorb the other layers.

## Desire-First Entry Model

A strong item entry should answer why someone in the world wants the item.

Core questions:

- Who wants it?
- Who already owns it?
- Who lost it?
- Who refuses to use it?
- What does it cost to keep?
- What changes if it disappears?
- What rumor makes people misunderstand it?
- Where on the map can it be found, forged, bought, stolen, awakened, or buried?

This approach makes items function as story engines instead of inventory decoration.

## Desire Triggers

Use the following triggers when drafting featured item entries.

| Trigger | Entry Use |
| --- | --- |
| Name hook | The name should create immediate curiosity or emotional color. |
| Visual silhouette | The reader should be able to imagine the object at a glance. |
| Origin | The object should feel born from a place, craft line, battle, faith, disaster, or bargain. |
| Reason for desire | Someone should want it for a clear reason. |
| Missing number or blank | Unfound pieces, missing owners, or incomplete sets create collection pull. |
| Set completion | Relic groups should make readers wonder what the full set does or reveals. |
| Growth or unlock | Some items become more meaningful through bond, trial, damage, repair, or awakening. |
| Ego or personality | An item may resist, advise, deceive, remember, or choose its bearer. |
| Cost or curse | Ownership should have a price when the item is powerful. |
| Location | Map placement turns the item into a journey. |
| Rumor | Public misunderstanding can be more useful than a clean fact sheet. |

## Long-Term Item Taxonomy

The item archive should organize candidates into a small number of durable buckets.

| Bucket | Function |
| --- | --- |
| General equipment | Ordinary or semi-special gear that supports character identity but does not require a featured entry. |
| Growth item | An object that changes through use, bond, awakening, repair, or sacrifice. |
| Ego item | An item with will, memory, personality, voice, preference, or chooser logic. |
| Scattered setting-book item | An item mentioned in lore that needs absorption into a future encyclopedia. |
| Legendary lineage group | A named group such as seven swords or eleven spears, where each member has separate identity and collective appeal. |
| Set gear | Multiple pieces designed to be completed, separated, inherited, stolen, or reassembled. |
| Collection treasure | Objects whose value comes from rarity, myth, number, location, or collector desire. |

## Legendary Group Handling

Legendary groups should not become a flat list of strongest weapons. Their value is in the pattern.

For a seven-sword or eleven-spear group, each object should ideally have:

- distinct name
- distinct silhouette
- origin region
- previous bearer or intended bearer
- known absence or current rumor
- emotional theme
- reason it belongs to the group
- reason it stands apart from the group

The setting book should leave useful blanks where the canon is not ready. Missing entries can become future story hooks.

## Trade Goods Versus Relics

Trade goods and relics should be separated even when both are desirable.

Trade goods answer:

- Who produces this?
- Who transports it?
- Who taxes it?
- Who steals it?
- Which city or route becomes important because of it?

Relics answer:

- Who made or awakened it?
- Who is allowed to bear it?
- What history does it preserve?
- What moral, political, or magical cost follows it?
- Why would factions fight over it?

Some items can cross layers. A rare material can be a trade good, then become the substance of a relic, then become a symbol of a faction. The setting book should show that chain only when the evidence supports it.

## Chapter Lock

This chapter locks item handling for the current setting-book recomposition pass:

- Do not promote every extracted candidate into canon.
- Confirm object status before entry writing.
- Separate equipment, trade goods, relics, materials, devices, and magic names.
- Write item entries around desire, story cost, ownership, and map placement.
- Preserve protected name collisions such as Aegis.
- Use legendary groups and set items to create collection pull without over-defining unfinished canon.
