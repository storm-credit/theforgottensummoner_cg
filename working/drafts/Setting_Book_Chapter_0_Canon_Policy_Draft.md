# Setting Book Chapter 0 - Canon Policy Draft

이 장은 설정집 전체를 읽는 기준을 먼저 고정한다.

`THE FORGOTTEN SUMMONER`의 설정은
멋있는 정보의 총량보다
인간 서사, 관계의 흔적, 액트마다 남는 감정적 결산을 우선한다.

## 0.1 Core Vision

이 세계의 중심은 신격전이나 세계 멸망형 초월전이 아니다.

가장 먼저 읽어야 할 것은 아래 다섯 가지다.

1. 인간 서사.
2. 성장하는 영웅군상.
3. 액트마다 끝나는 감정선.
4. 과거, 현재, 미래가 뒤섞이는 비대칭 관계.
5. 소문, 흔적, 스침으로 남는 인연.

설정집은 세계를 크게 보이게 하기 위한 문서가 아니라,
인물이 어떤 무대 위에서 선택하고 상처받고 바뀌는지를 설명하기 위한 문서다.

따라서 세력 문서는 권력표가 아니라 삶의 무대여야 하고,
인물 문서는 전기문이 아니라 관계의 결절점이어야 한다.

## 0.2 Tone Policy

표면 명칭은 정통 에픽 판타지 톤을 우선한다.

세력명, 지명, 가문명, 아이템명은
현대 기업물이나 범죄 스릴러의 직설 어휘보다
역사와 전승이 느껴지는 이름을 우선 검토한다.

사용하기 좋은 계열:

- 연맹
- 동맹
- 상회
- 교역단
- 연합
- 결사
- 기사단
- 학회
- 회랑

주의할 계열:

- 카르텔
- 메가코프
- 현대 행정기관처럼 들리는 직설명
- 동양풍, 무협풍, 강호/문파/내공 계열 어휘

기능상 `cartel-type economy bloc`처럼 읽히는 조직도
세계 안 표면명은 `금융 연맹`, `검은 상회`, `지하 교역 동맹` 같은 방향으로 조정한다.

## 0.3 Canon Tier

설정집은 모든 정보를 같은 확정도로 다루지 않는다.

### Hard Canon

지금 당장 뒤집기 어려운 기준이다.

예:

- 작품 철학.
- 인간 서사 중심.
- 액트 구조.
- `14 = 서사 중심 영웅`.
- `15 = Named Notables + Operational Lines`.
- 동양 / 무협 제외.
- 범대륙은 후기 확장.

### Soft Canon

유력하지만 조정 가능성이 남은 기준이다.

예:

- 세력 세부 위계.
- 내부 부서 구조.
- 정확한 지리 정보.
- 세부 연표.
- 세부 장비와 전승.
- 현재 `stable_15_workset`으로 관리되는 15 Named Notables 후보.

### Open Question

아직 결론을 내리지 못한 항목이다.

예:

- 상충하는 설정.
- 중복 이름과 변형 표기.
- 14/15 경계 인물.
- 어느 문서를 정본으로 삼을지 모호한 경우.
- 관계의 시점이 불명확한 경우.

### Deprecated / Legacy

구상 구버전, 백업, 폐기 후보, 깨진 루트, 중복 루트다.

삭제하지는 않지만,
자동으로 정본으로 복귀시키지 않는다.

## 0.4 Source Priority

충돌이 생기면 아래 순서로 읽는다.

1. Canon Spine Reference.
2. Imported Working Copy Snapshot.
3. Derived Analysis.
4. Legacy / Backup / Corrupted Root.
5. Rumor / In-World Text.

Canon Spine Reference는 가장 우선한다.

여기에는 `workflow`, `audit`에서 반복적으로 기준으로 쓰는 문서,
대륙 / 세력 / 14-15 라우팅의 기준 reference,
오케스트라가 `Hard Canon` 또는 `Active Spine`으로 판정한 문서가 포함된다.

Imported Working Copy Snapshot은 원본에서 복사해 온 작업본이다.
직접 수정하지 않고 근거 확인용으로만 사용한다.

Derived Analysis는 오케스트라가 원본과 작업본을 읽고 만든 분석 문서다.
실행 판단에는 중요하지만,
더 높은 출처 증거와 충돌하면 `Conflict` 또는 `Open Question`으로 낮춘다.

Legacy / Backup / Corrupted Root는 바로 정본 후보로 쓰지 않는다.

Rumor / In-World Text는 세계 안에 존재할 수 있지만,
사실과 동일하게 취급하지 않는다.

## 0.5 Archive Routing

새 정보가 들어오면 먼저 어디에 속하는지 판단한다.

| Route | Use |
|---|---|
| `8. 세력 아카이브` | 세력, 기관, 집단, 사회 구조, 장소 기반 조직 |
| `14. 영웅 백과` | 서사 중심 영웅, Act 중심 인물, 독립 인물 파일 |
| `15. Named Notables` | 세계 안에서 찾아갈 이유가 있는 이름 있는 명사형 인물 |
| `15. Operational Lines` | 조직 작동을 보여주는 실무층 |
| `아이템` | 장비, 유물, 소지품, 도감성 후보 |
| `지도 / 공간` | 장소 기능, 이동, 도시, 건물, 항로 |
| `hold` | 판단 보류, source check, dual review |

`15 Named Notables`는 약한 영웅 창고가 아니다.

현자, 장인, 기록자, 감정사, 연금술사, 공방주처럼
세계 안에서 이름이 알려진 명사형 인물을 회수하는 축이다.

반대로 `SS / S / A급`, Act 중심성, 전설 영웅록, 독립 14 파일 신호가 있으면
먼저 `verify_before_15`로 둔다.

## 0.6 Expansion Control

확장은 메인 축을 흔들지 않는 선에서만 허용한다.

설정집 작성 중 새 정보가 매력적으로 보여도,
아래 순서를 넘지 않는다.

1. source priority 확인.
2. canon tier 부여.
3. state label 부여.
4. revision gate 확인.
5. 본문 편입 또는 hold 처리.

판단이 안 되면 삭제보다 보류를 우선한다.

`Open Question`은 설정집 밖으로 밀어내는 실패가 아니라,
나중에 더 좋은 근거로 회수하기 위한 안전한 대기 상태다.

## 0.7 Story Handoff

설정집은 `무엇이 존재하는가`를 다룬다.

하지만 실제 소설은
`그것이 언제, 누구에게, 어떤 감정으로 드러나는가`를 다룬다.

따라서 설정집 본문은 아래 선을 지킨다.

- 장면 연출을 대신하지 않는다.
- 대화 리듬을 확정하지 않는다.
- 독자가 언제 알게 될지를 함부로 고정하지 않는다.
- 복선과 공개 타이밍은 `FS Story Engine` 쪽으로 넘긴다.

설정은 인물보다 앞서지 않는다.

설정집은 장면을 묶어 버리는 문서가 아니라,
장면이 더 강하게 작동하도록 기준을 정리하는 문서다.

## 0.8 Chapter Lock

이 장에서 고정하는 기준은 아래다.

- 작품 철학이 기존 문서보다 우선한다.
- 인간 서사와 관계 개연성이 세부 설정의 화려함보다 우선한다.
- 삭제보다 보류를 우선한다.
- legacy는 보존하되 자동 복귀시키지 않는다.
- `14`는 서사 중심 영웅축이다.
- `15`는 Named Notables와 Operational Lines의 이중 레이어다.
- 원본 snapshot은 근거 확인용이고, 직접 수정 대상이 아니다.
