# Setting Book Chapter 4 Draft: Naming Normalization and Display Policy

## Chapter Purpose

This chapter turns the naming cleanup work into a setting-book-facing policy. Its role is not to rename every source file at once. Its role is to explain which names are preferred for display, which labels remain internal, and which similar-looking names must stay separated until direct evidence proves otherwise.

The setting book should treat names as part of world tone. A name is not only an identifier. It signals continent, faction culture, social weight, history, and genre.

## Core Naming Rule

Display canon and internal audit labels are different layers.

- Display canon names are the names readers, setting-book entries, and future story-facing material should normally see.
- Internal audit labels are allowed to remain functional, blunt, or provisional if they help tracking.
- A working label must not be promoted into display canon just because it appears often in audit material.
- A `_Legacy_` path or old extracted file is not final canon evidence by itself.
- If a file path and body text disagree, the file path is treated as the first location clue, not as automatic truth.
- Conflicts should be collected before mass correction.

## Genre Tone Policy

The naming palette should support a human-centered orthodox epic fantasy tone. It should feel broad enough for continent-scale politics, trade, faith, magic, bloodline history, and relic myth without sounding like modern corporate crime fiction or East Asian martial-world terminology.

Preferred surface vocabulary includes:

- league
- alliance
- union
- concord
- guild
- company
- trade house
- caravan
- conclave
- order
- academy
- society
- council
- chapter

Terms requiring review before display use:

- cartel
- syndicate
- megacorp
- corporation
- industry
- gangland-style labels
- murim, gangho, inner-power, or cultivation-coded wording

These terms may remain as internal analysis categories, but the setting-book display layer should usually convert them into fantasy-facing names unless a specific faction is intentionally modern, criminal, or satirical.

## Name Collision Guard

Similar names must not be merged merely because they look close. The setting currently contains repeated family names, titles, relic names, magic names, and extraction variants. Each case needs source support before consolidation.

### Sylvia Cluster

The following are separate candidates unless a direct source proves identity overlap:

- Kirke Sylvia
- Sylvia Arkham
- Oceanic Sylvia
- Sylvia Phantom

Do not collapse them into one Sylvia entry. The setting book may include a collision note if multiple Sylvias remain relevant.

### Aegis Cluster

`Aegis` appears across person, surname, item, magic, and defensive concept layers. The setting book should treat it as a protected collision term.

Allowed distinctions:

- person or family name
- relic or item name
- shield magic or defensive doctrine
- symbolic title

No Aegis entry should absorb another Aegis layer without evidence.

### Ravenfell and Related Marcus Names

Marcus Ravenfell, Maxwell Ravenfell, and Marcus Corvus should remain distinct unless the source trail confirms a deliberate alias, inheritance, or family-role relationship.

### Mera Cluster

Mera Lylewind and Mera Sylphid should not be merged by name alone. If both remain active, the setting book should identify them by faction, continent, role, or source path.

### Rafferty and Raphael Arcadia

Rafferty and Raphael Arcadia remain unresolved as either name drift, alias, translation variation, or duplicate candidate. Do not promote a single form until the source relationship is resolved.

### Other Protected Similar-Name Areas

The same non-merge rule applies to Valerius, Drake Ruga or Drake Rawson, Selena, Sylas, Wolfgar, Erion, Christopher Delmar, Isador or Isadore variants, and any same-name candidates appearing across different continents or factions.

## Setting-Book Entry Pattern

When a name has unresolved variants, use this pattern:

1. Lead with the current display name.
2. Add variant forms in a short note.
3. State whether the variants are spelling drift, translation drift, alias candidates, or protected collision candidates.
4. Avoid rewriting story logic around the variant until the evidence is stable.

Example:

```text
Spirit Union
Variant forms: Elemental Alliance, 정령 연합.
Status: English display normalized to Spirit Union. Korean axis remains 정령 연합.
```

## Chapter Lock

This chapter locks naming behavior for the current setting-book recomposition pass:

- Use display names in prose.
- Keep internal labels out of reader-facing sections unless needed as variant notes.
- Replace modern crime or corporate vocabulary with fantasy-facing faction language where a replacement candidate exists.
- Protect same-name clusters from accidental merge.
- Treat corrupted mixed-language fragments as cleanup targets, not as lore.
