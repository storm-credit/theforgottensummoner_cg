# FS Engine

## Definition

`FS Engine`은
아스트라리스 설정집을 역설계하고 정리할 때 쓰는
오케스트라의 내부 판단 엔진이다.

여기서 `FS`는
`Forgotten Summoner` 작업 전용 엔진 이름으로 쓴다.

이 엔진은 아래를 통합한다.

- 역설계 구조화
- 작가 방법론
- 관계망 설계
- 개연성 감사
- 확장 통제
- 명칭 톤 제어

즉 FS 엔진은
`설정 운영 엔진`이면서 동시에
`작법 엔진`이다.

즉 `작법 메모`가 아니라
오케스트라가 실제로 어떤 렌즈를 먼저 적용할지 정하는
실행 엔진이다.

운용상 FS 엔진은 아래 세 하위 엔진으로 나누어 쓴다.

- `workflow/15_FS_Lore_Engine.md`
- `workflow/16_FS_Story_Engine.md`
- `workflow/14_FS_Media_Engine.md`

현재 설정집 정리 단계에서는
`FS Lore Engine`을 기본 주 엔진으로 쓴다.

## Engine Philosophy

FS 엔진은 특정 작가 한 명의 공식이나
특정 이론 하나를 절대화하지 않는다.

대신 여러 방법론을
`역할별 모듈`로 분해해 재조립한다.

핵심 원칙:

1. 인간 서사가 중심이다.
2. 관계망이 구조보다 약하면 실패로 본다.
3. 멋있는 설정보다 작동하는 설정을 우선한다.
4. 확장은 메인 축을 흔들지 않는 선에서만 허용한다.

## Method Modules

### 1. Reverse Design Module

역할:

- 이미 흩어진 설정과 문서를 보고
  나중에 `정본 구조`를 거꾸로 복원한다.
- 현재 문서가 어떤 상위 구조를 암시하는지 읽어낸다.

주 사용처:

- `8번` 구조 안정화
- `14번`과 `15번` 분리
- 레거시 / 고아 문서 판정

### 2. Story Architecture Module

역할:

- 세계관, 대륙, 세력, 인물, 욕망, 충돌을
  하나의 큰 서사 골격으로 본다.

주 사용처:

- 세계관 코어
- 대륙 프레임
- 세력 spine

### 3. Conflict Module

역할:

- 누가 왜 충돌하는지,
  같은 사실을 왜 다르게 해석하는지,
  왜 관계가 비틀리는지를 본다.

주 사용처:

- 세력 대립
- 관계 타입
- 같은 편이었다가 엇갈리는 구조

### 4. Character Arc Module

역할:

- 인물이 무엇을 믿고 잃고 바꾸는지 본다.
- 강함보다 감정 비용과 변화 방향을 우선한다.

주 사용처:

- `14번` 중심 영웅
- `15번` Named Notables와 중간 인물의 서사 가치 판정

### 5. Relationship Network Module

역할:

- 인물들을 그래프로 보고
  중심성, 고립, 매개, 군집을 점검한다.

주 사용처:

- 관계망 표
- 인물-조직 접점
- 과연결 경고

### 6. Plausibility Module

역할:

- 이 설정이 실제 장면과 결과로 이어지는지 본다.
- 설정이 화려해도 순리가 없으면 경고한다.

주 사용처:

- 개연성 감사
- 액트별 정서 비용 점검
- 도시, 세력, 아이템의 실사용성 판정

### 7. Expansion Control Module

역할:

- 설정이 커질 때
  핵심 줄기를 망가뜨리지 않도록 통제한다.

주 사용처:

- 범대륙 후기 확장
- 아이템 소진행
- 지도 점진 확장
- 생활상 / 문화 / 예술 확장

### 8. Naming Tone Module

역할:

- 구조 라벨과 세계 안 표면 명칭을 분리해 관리한다.
- 서양 정통 에픽 판타지 톤을 유지한다.

주 사용처:

- 세력명
- 직능명
- `15번` Operational Lines display canon
- Named Notables 명칭 정리

### 9. Canon Tier Module

역할:

- 모든 항목을 `Hard / Soft / Open Question / Deprecated`로 분류한다.
- 충돌 발견 시 바로 수정하지 않고
  먼저 티어를 판정한다.

주 사용처:

- 충돌 문서
- 보류 문서
- 14/15 경계 인물

### 10. Archive Routing Module

역할:

- 새 정보가 어디로 가야 하는지 판정한다.

주 사용처:

- `14 Hero Archive`
- `15 Named Notables`
- `15 Operational Lines`
- `8 Faction Archive`
- `hold / legacy`

### 11. Chronology / Causality Module

역할:

- 사건, 이동, 만남, 소문, 부재를 시간축 위에 정렬한다.

주 사용처:

- 액트 정리
- 재회 순리 검사
- 시점 충돌 탐지

### 12. Act Outcome Ledger Module

역할:

- 액트가 끝날 때 실제로 무엇이 바뀌었는지 남긴다.

주 사용처:

- 관계 변화
- 상실 / 획득
- 다음 액트 연결

### 13. Foreshadow / Payoff Module

역할:

- 복선을 객체 단위로 등록하고 회수 상태를 추적한다.

주 사용처:

- 복선 심기
- 회수 누락 방지
- 인물/유물/약속/상처 관리

### 14. Legacy Quarantine Module

역할:

- `_Legacy_`, `Backup`, 폐기 초안, 흔들리는 정본 후보를 격리한다.

주 사용처:

- 레거시 정리
- 중복 구조 보류
- 자동 인용 방지

### 15. Decision / Ruling Module

역할:

- Conductor가 내린 핵심 판정을 짧게 남긴다.
- 같은 문제를 다시 처음부터 논의하는 비용을 줄인다.

주 사용처:

- 14/15 경계 판정
- 이름 충돌
- deferred expansion
- grade_caution 기록

### 16. Cross-Chronicle Firewall Module

역할:

- 다른 크로니클의 톤, 어휘, 서사 문법이
  아스트라리스 판타지 버전으로 새지 않게 막는다.

주 사용처:

- 동양/무협 차단
- 현대 직설 용어 완화
- 과열된 초월 서사 차단

### 17. Slot Maturation Module

역할:

- `need_named_candidate` 슬롯이
  언제 실제 named notable 후보로 자라는지 추적한다.

주 사용처:

- 프로스트 / 해양 / 에테르의 이름 없는 명사층
- display canon 후보에서 실제 인명 후보로 올라가는 과정

### 14번 영웅백과

1. Character Arc Module
2. Relationship Network Module

### 15번 Named Notables

1. Character Arc Module
2. Plausibility Module
3. Naming Tone Module
4. Archive Routing Module
5. Canon Tier Module

질문:

- 이 인물은 세계 안에서 `찾아갈 이유`가 있는가
- 영웅은 아니어도 `기억되는 얼굴`인가
- 장인, 현자, 기록자, 연금술사, 항해사 같은 명사형 가치가 있는가
- `SS / S / A급`, `Act 중심성`, `전설 영웅록`, `독립 14 파일` 신호가 강하지 않은가
- 직업별 분류가 아니라 `대륙 -> 세력 / 도시 / 조직` 앵커로 라우팅되는가

## Orchestrator Rule

오케스트라는 FS 엔진을 이렇게 쓴다.

1. 지금 정리 대상이 무엇인지 본다.
2. FS 엔진 안에서 먼저 적용할 모듈을 고른다.
3. 필요한 경우 여러 모듈을 순차 적용한다.
4. 결과를 `14`, `15 Named Notables`, `15 Operational Lines`, `8`, `지도`, `아이템` 중 어디에 떨어뜨릴지 정한다.
5. 충돌 시 먼저 `Canon Tier`와 `Archive Routing`을 적용한다.

즉 오케스트라는
`적절하게 엔진 안의 방법론을 꺼내 쓰는 지휘자`다.

## Current Use

지금 오케스트라는 아래처럼 FS 엔진을 실제 적용 중이다.

- `15 Named Notables`:
  - `에리온 드라코비스`, `울프가르`, `오그마`, `실비아` 같은 후보를
    `Character Arc + Plausibility + Naming Tone`으로 보고 있다.
- `15 Operational Lines`:
  - 항만, 경매, 첩보, 금융, 용병, 위조 라인을
    `Reverse Design + Plausibility + Naming Tone`으로 보고 있다.
- `8번 세력 구조`:
  - `Reverse Design + Story Architecture + Conflict`
- `14번 영웅축`:
  - `Character Arc + Relationship Network`
