# Lore Audit Rules

## 감사 대상

모든 이슈는 아래 네 범주로 나눈다.

1. `Gap`
2. `Conflict`
3. `Duplicate`
4. `Structure`

## 공통 감사 질문

- 이 문서는 인간 서사를 밀어주나, 설명을 과하게 늘리나
- 이 정보는 관계와 액트에 실제로 쓰일 가능성이 있나
- 이 문서는 상위 문서와 실제 연결되나
- 같은 대상이 다른 이름이나 다른 분류로 중복되나
- 이 문서가 정본인지 레거시인지 판단 가능한가

## 8. 세력 아카이브 전용 규칙

- 세력은 먼저 `무대`, `역할`, `주요 인물 접점`이 보여야 한다.
- 대륙별로 `가문 / 부족 / 길드` 층이 비어 있지 않은지 먼저 본다.
- 문화, 경제, 예술, 식생활 확장 문서는 루트 구조가 안정된 뒤에 정리한다.
- 대륙 루트와 세력 루트가 일관되게 이어져야 한다.
- 깨진 폴더명, 중복 대륙 폴더, `_Legacy_` 구조를 우선 추적한다.

## 14. 인물 백과 전용 규칙

- 인물 문서는 능력보다 관계와 서사 비용을 먼저 본다.
- 성장 영웅, 현존 영웅, 소환 영웅의 구분이 실제 쓰임과 맞는지 본다.
- 인물명, 세력명, 대륙명이 `8번` 문서와 연결 가능한지 확인한다.
- 인물 하나가 지나치게 많은 세력과 사건을 독점하면 경고한다.

## 우선순위

### P0

- 루트 구조 파손
- 중복 대륙 또는 중복 세력 폴더
- 문서 경로가 실제 분류 체계와 맞지 않는 경우

### P1

- 세력명 또는 인물명 표기 흔들림
- 레거시와 현행 구조 충돌
- 주요 인물 접점 문서 누락

### P2

- 과도한 확장 문서
- 서사보다 설명이 앞서는 문서
- 후순위 세부 설정의 과밀

## 보고 형식

각 감사 결과는 다음 단위를 유지한다.

- 대상 경로
- 이슈 분류
- 심각도
- 왜 문제인지
- 당장 고칠지, 보류할지
