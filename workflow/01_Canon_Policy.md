# Canon Policy

## Canon 등급

### Hard Canon

지금 당장 뒤집기 어려운 기준이다.

- 작품 철학
- 인물 중심의 서사 방향
- 액트 구조의 존재
- 특정 인물군의 핵심 역할
- 연대기 간 큰 결

### Soft Canon

유력하지만 조정 가능한 기준이다.

- 세력 세부 위계
- 내부 부서 구조
- 정확한 지리 정보
- 세부 연표
- 세부 장비와 전승

### Open Question

결정하지 않은 항목이다.

- 상충하는 설정
- 중복 이름과 변형 표기
- 어느 문서를 정본으로 삼을지 모호한 경우
- 관계의 시점이 불명확한 경우

## 우선순위 규칙

1. 작품 철학이 기존 문서보다 우선한다.
2. 인간 서사와 관계 개연성이 세부 설정의 화려함보다 우선한다.
3. 같은 정보가 충돌하면 더 넓은 맥락과 연결된 문서를 우선 검토한다.
4. 삭제보다 보류를 우선한다.
5. `_Legacy_`, `Backup`, 참조용 파일은 정본 후보에서 제외한다.

## 섹션별 정본 판단

### 8. 세력 아카이브

- 정본 후보: 대륙 루트, 세력 루트, 하위 장소와 예하 조직의 실제 문서
- 보류 후보: 문화, 경제, 예술, 생활상처럼 확장 문서가 과도한 경우
- 삭제 후보 아님: 레거시 폴더 안의 문서. 먼저 분류 후 보관 여부 결정

### 14. 인물 백과

- 정본 후보: 인물 개별 파일
- 보류 후보: 중복 이름, 다른 세력 표기, 다른 관계 정의
- 후순위: 세부 장비, 수치, 전투력, 과장된 설정

### 15. 인물 백과 예정 축

- 역할: 다른 문서에 흩어진 중간 등장인물 수집
- 원칙: `14번`에 있는 인물을 `15번`으로 이동하지 않는다
- 대상: 세력 문서, 지역 문서, 사건 문서, 기타 백과에 흩어진 보조 인물

### 아이템 백과

- 역할: 문서마다 흩어진 장비, 유물, 소지품, 특수 도구를 수집해 정리
- 원칙: 인물 문서와 세력 문서에서 바로 확정하지 않고 먼저 수집 후보로 등록
- 후순위 아님: 흩어진 정보 회수는 지금부터 진행하되, 정본 편입은 구조 정리 뒤에 한다

## 기록 방식

- 확정은 `Hard Canon`
- 유력은 `Soft Canon`
- 모순은 `Conflict`
- 누락은 `Gap`
- 구조 문제는 `Structure`

## Naming Policy

- 내부 기능 분류와 세계 내 표면 명칭은 분리할 수 있다.
- 예를 들어 내부적으로는 `cartel-type economy bloc`로 읽더라도, 표면 정본명은 `금융 연맹`, `검은 상회`, `지하 교역 동맹` 같은 판타지 톤을 우선한다.
- 세력명은 현대 경영 용어보다 정통 판타지 질감, 대륙별 어계, 역사적 뉘앙스를 우선한다.
- 인물명도 대륙별 음운 규칙 안에서 관리하고, 동양/무협 계열 어감은 이 크로니클에서 제외한다.
