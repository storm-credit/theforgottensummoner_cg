# Five Continent Missing Layer Master Lock

이 문서는
`결손층 5개` 본선을 한 장에서 읽기 위한 master lock이다.

목적:

- 흩어진 `policy / register / first pass / second pass / firewall`을
  한 입구에서 다시 읽을 수 있게 한다.
- 이후 설정집 본선 보강은 이 문서가 가리키는 다섯 기준을
  한 세트로 유지하는 방식으로만 진행한다.
- `새 층 발명`이 아니라
  `thin/support 정책 유지 + admissible evidence intake + overread 차단`
  순서로만 보강하게 잠근다.

## Core Set

이 master lock이 직접 묶는 기준 문서는 아래 다섯 개다.

1. `audit/Five_Continent_Missing_Layer_Policy_Lock.md`
2. `audit/Five_Continent_Missing_Layer_Evidence_Register.md`
3. `audit/Five_Continent_Missing_Layer_Evidence_First_Pass_A.md`
4. `audit/Five_Continent_Missing_Layer_Evidence_Second_Pass_A.md`
5. `audit/Five_Continent_Missing_Layer_Overread_Firewall.md`

## Locked Reading Order

결손층 5개는 아래 순서로 읽는다.

1. `Policy Lock`
   - 각 대륙에서 얇은 층을 어디까지 허용하는지 본다.
2. `Evidence Register`
   - 어떤 신호를 admissible로 받을 수 있고 무엇을 reject하는지 본다.
3. `First Pass A`
   - 실제 import 본문에서 1차 샘플 anchor가 어디인지 본다.
4. `Second Pass A`
   - 같은 방향의 반복 증거가 다른 문서 family에서도 재확인되는지 본다.
5. `Overread Firewall`
   - tempting signal을 왜 승격 근거로 못 쓰는지 최종 차단선을 본다.

## Continent Lock Summary

| Continent | Thin/Support Lock | Stable Safe Read | Main Overread Ban |
|---|---|---|---|
| `에테르` | outside `Spirit Union` 부족층은 경계 생존/완충 실무까지만 받는다 | `frontier echo / border support` | outside core를 곧바로 부족 연합 본체로 복원 |
| `크림슨` | 씨족형 지배층 위의 얇은 국가형 가문층만 허용한다 | `tribe_clan core + state_house thin-support` | 세습 귀족국가 / `state_house strong` 복원 |
| `프로스트` | 정주 귀족층은 요새/보급/병기소 운영층 support로만 읽는다 | `settled stronghold support layer only` | settled nobility enough 승격 |
| `해양` | 토착 공동체층은 fleet/church/port 바깥 local support까지만 보존한다 | `local support outside fleet/church/port core` | tribe_clan enough 승격 / fleet-core를 토착층으로 오독 |
| `오벨리스크` | state-house signal은 먼저 비정통 엘리트로 우회시킨다 | `state_house signal rerouted to nontraditional elite first` | monarchy / traditional state-house 복원 |

## Conductor Rule

1. 결손층 보강은 이 다섯 문서를 한 세트로 읽을 때만 진행한다.
2. 새 evidence가 들어오면 먼저 `register`에 admissible class가 있는지 본다.
3. admissible class에 들어와도 즉시 승격하지 않고 `first/second pass` 방향과 같은지 본다.
4. 마지막으로 `firewall`을 통과하는지 확인한 뒤에만 safe read를 보강한다.
5. `summary / bridge / watch / profile / place-pressure` 문서는 이 master lock을 넘어서 대륙 본체를 재정의하지 않는다.

## Conductor Decision

`결손층 5개`는 이제
`정책`,
`증거 intake`,
`1차 샘플`,
`반복 증거 안정화`,
`overread 차단`
까지 한 세트로 닫혔다.

즉 다음 본선은
빈칸을 새로 만드는 일이 아니라,
이 master lock을 기준으로만
새 evidence를 받고 거르는 작업으로 읽는다.
