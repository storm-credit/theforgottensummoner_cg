# 절대 보안 암살 청부 접수자 계열

## 1. Core Identity

- primary anchor:
  - `침묵의 상회`
- role:
  - `암살 청부 접수 창구 관리자`
- type:
  - `Organization Mesh`
- status:
  - `safe_15`

## 2. World Function

- 누가 죽어야 하는지보다
  `누가 그 죽음을 접수하고 검증하는가`를 보여주는 계열이다.
- 침묵의 상회를 단순 살해 집단이 아니라
  의뢰, 검증, 봉인, 전달 절차를 가진 조직으로 보이게 만든다.
- 영웅과 직접 싸우지 않아도
  세계의 많은 비극이 이 창구를 통과했을 수 있다는 점에서 존재감이 크다.

## 3. Spatial Tie

- primary place:
  - `봉인 접수실`
- secondary place:
  - `암살 의뢰 보관고`
- route value:
  - `접수실 / 보관고 / 봉인문서 선반 / 사전 검증실`

## 4. Relation Hooks

- `사령 기반 텔레파시망 관리자 계열`
  - `ally`
  - 봉인된 의뢰가 죽은 통신망을 통해 전달될 수 있다.
- `네크로 우물 현장 연락책 계열`
  - `ally`
  - 현장 접선과 청부 전달이 연결된다.
- `환상 암살단 현장 집행자 계열`
  - `ally`
  - 접수된 의뢰가 실제 살해 집행으로 이어진다.
- `배신 귀족 / 복수자 / 절망한 상단`
  - `intermittent_alignment`
  - 의뢰자이면서도 언제든 제거 대상이 될 수 있는 고객층이다.

## 5. Archive Use

- 침묵의 상회 묶음에서
  `의뢰가 조직으로 들어오는 입구`를 맡는다.
- 봉인 접수실과 보관고 같은 내부 공간 서사에 강하다.
- 청부, 배신, 거절된 의뢰, 사라진 봉인문서 같은 서사에 유용하다.

## 6. Conductor Note

- 지금 확정할 것:
  - 암살 청부의 접수와 검증을 담당하는 계열이라는 점
  - 봉인 접수실과 의뢰 보관고에 강하게 묶인다는 점
  - 침묵의 상회 운영 절차를 보여주는 핵심 창구라는 점
